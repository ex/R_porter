Como usar scripts con R_porter 1.1 (Alpha 2)

04 Octubre 2004

¿Nunca has necesitado renombrar 4000 archivos JPG? Hace poco tuve que hacerlo pero antes de hacerlo a mano prefiero agregarle esta característica a mi programa.

R_porter puede cargar un archivo script (vbs) el cual será llamado a medida que se realiza la exploración de un directorio o unidad. Como ejemplo solo voy a mostrar como alterar el reporte por defecto del programa, pero existe completa libertad para lo que puedes hacer con tu script.

 

  1. Primero en [Opciones Avanzadas] activa la casilla [Activar scripts durante exploración], luego haz click en el botón de la derecha para seleccionar el script a usar.



  2. Selecciona el script (en nuestro caso el script de ejemplo que viene con el instalador). Dale click al botón de [Aceptar].



  3. Si quieres analizar este script usa el menu [Archivo]->[Script] del programa principal, te saldrá un pequeño editor de texto, selecciona el menu para cargar el script como se muestra



  4. Este editor es bastante ineficiente y no lo recomiendo para editar grandes cantidades de código. (Pensaba usarlo sólo para editar pequeñas sentencias SQL). Analiza el código para que te deas cuenta de donde debes cambiarlo para acomodarlo a tus necesidades. Está comentado.


  5. No es necesario que el script este abierto para que se ejecute. Selecciona una carpeta cualquiera e inicia la exploración. Verás como el script es llamado en cada etapa de la exploración. En el caso del ejemplo hemos puesto código para anular el reporte por defecto del programa si se responde con [Si] al mensaje.



  6. El script de ejemplo sólo enumera los archivos y directorios encontrados. Pero como verás no es muy dificil crear rutinas más complejas. Lo dejo a tu imaginación.
